# EyeDropper API

## 개요
EyeDropper API는 사용자가 화면 위의 임의의 픽셀에서 색상을 선택할 수 있는 기능을 제공합니다. 클릭하면 스크린 전체에 대한 색상 샘플링 모드가 활성화되어, 사용자가 원하는 색상을 클릭하면 해당 픽셀의 sRGB 색상 값을 반환합니다. 주로 색상 조정 도구나 디자인 앱에서 활용됩니다.

## 권한
필요한 권한 없음

## 폴더 구조
```
eyeDropper/
├── index.html          # 앱의 진입점, 색상 선택 버튼과 색상 표시 영역의 기본 레이아웃 제공
└── src/
    ├── main.js         # EyeDropper API 호출 및 색상 표시 로직 구현
    └── style.css       # 색상 표시 영역, 버튼 등의 스타일 정의
```

## 파일 설명

### index.html
앱의 진입점으로, 색상 선택 버튼, 색상 표시 영역, 선택된 색상 텍스트를 표시할 레이아웃을 구성합니다. `src/main.js`와 `src/style.css`를 로드합니다.

### src/main.js
EyeDropper API를 사용하여 색상 선택 기능을 구현하는 핵심 파일입니다.

- **색상 선택 버튼 클릭 처리**
  - 버튼이 클릭되면 먼저 `window.EyeDropper`의 지원 여부를 확인합니다.
  - 지원되면 `new EyeDropper()`로 인스턴스를 생성한 후 `eyeDropper.open()`을 호출합니다. `open()`은 비동기 메서드로, 사용자가 색상을 선택할 때까지 대기합니다.
  - 사용자가 색상을 선택하면 반환된 `result.sRGBHex` 값을 사용하여 색상 표시 영역의 `backgroundColor`와 텍스트를 업데이트합니다.

- **색상 선택 취소 처리**
  - 사용자가 색상 선택을 취소하면(예: Esc 키 입력) `catch` 블록이 실행되어 "선택 취소됨" 메시지를 표시합니다.

### src/style.css
색상 선택 버튼, 색상 표시 영역, 선택된 색상 텍스트의 스타일을 정의합니다.

## 동작 순서
1. 앱이 로드되면 색상 선택 버튼, 색상 표시 영역, 텍스트 영역이 렌더링됩니다.
2. 사용자가 색상 선택 버튼을 클릭합니다.
3. `window.EyeDropper`의 지원 여부가 확인됩니다. 미지원 시 적절한 안내가 표시됩니다.
4. 지원되면 `new EyeDropper()`로 인스턴스가 생성되고, `eyeDropper.open()`이 호출되어 스크린 전체 색상 샘플링 모드가 활성화됩니다.
5. 사용자가 원하는 색상의 픽셀을 클릭하면 `open()`이 해당 픽셀의 색상 값을 `result.sRGBHex` 형태로 반환합니다.
6. 반환된 색상 값으로 색상 표시 영역의 배경색과 텍스트가 업데이트됩니다.
7. 사용자가 색상 선택을 취소하면 `catch` 블록에서 "선택 취소됨" 메시지가 표시됩니다.

> **참고:** Chrome 95 이상에서만 지원됩니다.
